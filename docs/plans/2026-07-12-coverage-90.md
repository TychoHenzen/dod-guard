## Decisions (locked with user)

<decisions>
## Design Decisions

1. **Coverage measurement**: Node.js built-in --experimental-test-coverage for immediate feedback; c8 for CI/scripting
2. **CI gate**: Separate CI step after tests, checks per-file thresholds
3. **OS-dependent code**: Mark untestable paths with c8 ignore comments + document in skip_reasons
4. **Test framework**: Node.js native test runner — no new dependencies
5. **Mock strategy**: mock.module for ESM modules, mock.method for specific functions
6. **Coverage target**: 90% statements per file (not per package aggregate)
7. **TDD approach**: Red phase first — write test, verify it fails, then implement
</decisions>
